Automação (Arduíno)

Aprendi a usar um Arduino com display LCD, conectá-lo e configurá-lo para exibir Notícias e dados dinâmicos. Construa um semáforo que controle LEDs com lógica sequencial. Aprendi sobre o ESP32, configurei-o para conectividade Wi-Fi e explorei suas capacidades em projetos de IoT. Aprendi a programação básica desses microcontroladores e ampliei meus conhecimentos em eletrônica e automação. Arduino é uma plataforma de prototipagem eletrônica de código aberto baseada em hardware e software fáceis de usar. Consiste em um microcontrolador que pode ser programado para interagir com o ambiente por meio de sensores, atuadores e outros componentes eletrônicos. As placas Arduino são amplamente utilizadas em projetos de automação, robótica e Internet das Coisas (IoT).

A programação das placas Arduino é feita utilizando a linguagem de programação Arduino (semelhante ao C++) e o Arduino IDE, que permite escrever, compilar e enviar código para o microcontrolador. Além de sua facilidade de uso, o Arduino é popular pela grande comunidade de desenvolvedore e extensa documentação e exemplos online. As placas Arduino vêm em uma variedade de modelos, como Arduino Uno, Nano e Mega, cada um adequado para diferentes tipos de projetos e necessidades.

O que é esse ESP32? É um microcontrolador ideal para projetos IoT e possui processador dual-core, Wi-Fi e Bluetooth . Possui vários GPIOs e periféricos como ADC, DAC, PWM, UART, SPI e I2C. Pode ser programado em IDE Arduino, facilita a prototipagem rápida. É altamente eficiente em termos energéticos e adequado para as seguintes aplicações É necessária uma vida útil mais longa da bateria. Seus usos incluem automação residencial, monitoramento remoto, wearables e tecnologia robótica. Criar um semáforo usando Arduino é muito fácil, você vai precisar de um Arduino Uno, 3 LEDs (vermelho, amarelo, verde), 3 resistores de 220 ohms, jumpers e uma protoboard, tanto que fizemos um projeto bem simples desde o início Esses anos.

Banco de Dados

Ao apresentar o banco de dados, primeiro entendemos a modelagem e a estrutura dos dados, ou seja, os dados Dados estruturados, não estruturados e semiestruturados. A modelagem de dados envolve três etapas principais: Conceitos, lógica e física. Modelos conceituais descrevem entidades e seus relacionamentos de forma abstrata acabando por ser denpendente de qualquer sistema de gerenciamento de banco de dados. O modelo lógico transforma o modelo conceitual em Representação mais detalhada, incluindo tabelas, chaves primárias e chaves estrangeiras. Finalmente, a definição do modelo físico Como os dados são armazenados no banco de dados, especificando detalhes como tipos de dados, índices e partições. Esses três modelos fornecem uma estrutura abrangente para projetar e implementar sistemas de banco de dados eficientes. e se organize.

Aprendemos também sobre os Tipos De Armazenamento, o que seria um servidor.. tipos como Privados, Públicos e Hibridos. Os bancos de dados relacionais são sistemas de gerenciamento de dados que organizam informações em tabelas com linhas e colunas. Eles seguem o modelo relacional, que permite estabelecer relações entre essas tabelas por meio de chaves primárias e estrangeiras. A linguagem SQL é comumente usada para consultar e manipular dados em bancos de dados relacionais. Esses sistemas garantem integridade e consistência dos dados por meio de restrições e transações. Bancos de dados relacionais oferecem flexibilidade e escalabilidade para lidar com uma variedade de aplicações, desde pequenos projetos até grandes empresas. Eles são amplamente adotados devido à sua confiabilidade, desempenho e suporte a padrões industriais. Existem modelos de negocios e usamos 2 que seria o Modelo De Negocio 1: Modelo De Entidade Relacionamento (ER) é uma técnica de modelagem de dados que representa entidades como objetos distintos com atributos e relacionamentos entre eles. Entidades são representadas por retângulos, atributos por elipses e relacionamentos por losangos. Chaves primárias identificam exclusivamente cada entidade, enquanto chaves estrangeiras estabelecem conexões entre entidades. O ER é amplamente utilizado para projetar esquemas de banco de dados antes da implementação física. Enquanto o Modelo de Negocio 2: Modelo Conceitual - Diagrama ER descreve as entidades, seus atributos e os relacionamentos entre elas de forma abstrata. Utiliza retângulos para entidades, elipses para atributos e losangos para relacionamentos. É uma ferramenta visual eficaz na fase inicial de projeto de banco de dados, fornecendo uma visão clara das estruturas e interações dos dados.

Os Bancos de Dados NoSQL são sistemas de gerenciamento de dados que oferecem flexibilidade para lidar com grandes volumes de dados não estruturados ou semiestruturados. Eles se diferenciam dos bancos de dados relacionais ao não seguir um esquema fixo de tabelas, permitindo a inclusão de novos tipos de dados sem alterações no modelo. Existem diversos tipos relacionados a Banco de Dados NoSQL como: Chave Valor, que permitem as aplicações guardarem seus dados sem um formato especifico, e acabam sendo armazendas como duplas de chaves. Orientado a documento modelo que acaba armazenando coleções e seus documentos, os documentos meio que tem padrões de estrutura como o JSON, mas temos uma ausência de um esquema pré-definido. Familia De Colunas baseados em colunas guardam seus dados divididos por colunas,permitindo que sejam facilmente acessadas as colunas de múltiplos registros assim diminuindo o acesso ao disco e o tempo de processamento. Orientado a grafos, Modelo útil quando é necessário fazer consultas complexas, tais como mapas rodoviários até topologia de rede.

Java Orientação a Objetos

A linguagem de programação pode ser considerada padronizada, que usam regras ja semânticas para a implantação, ele também pode ser compilado para transformar em um programa do meu computador, aprendemos também sobre os compiladores, e interpretadores que geram arquivos para executar direto no sistema. Vimos também sobre variáveis variável é um espaço de armazenamento que pode conter diferentes valores ao longo do tempo durante a execução de um programa, e também fizemos 6 listas de exercicio relacionado a java.

As Estruturas de decisões que executam os if/else, permite que um programa execute diferentes ações com base em condições específicas, Estrutura de decisão simples if: Permite que o programa execute um bloco de código se uma condição for verdadeira, Estrutura de decisão composta if-else: Permite que o programa execute um bloco de código se uma condição for verdadeira e outro bloco de código se a condição for falsa. A Estrutra de decisão Switch também permite que os fluxos do programa sejam desviados para caminhos diversos, ou seja caso não for correspondido a nenhuma opção, o bloco vai ser o default e será executado. A estrutura de repetição while permite a criação de um loop que executa um bloco de instruções repetidas vezes enquanto uma condição for verdadeira, as funções são executas apenas quando chamadas e tem seus parâmetros que seria seus valores chamados quando solicitados.
